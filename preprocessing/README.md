# 自動前處理管線 v2（拍照原圖 → 論文構圖 512×512）

現場拍攝的「未裁切」感壓紙照片**不可直接餵給檢測器**：壓痕非主體、含藍／青色
背景紙、白平衡偏冷時，構圖與白平衡皆不符演算法前提（背景會被誤算為接觸、
壓痕紅度過不了門檻），指標會嚴重失真。

本資料夾提供與**論文 78 張手動裁切構圖對齊**的全自動前處理（Python）：

## 管線步驟（`crop_and_pretest_v2.py`）

1. **EXIF 方向校正**
2. **HSV 紙偵測**（青色色相 75–115、飽和度 >60，抗白平衡漂移；合併 ≥ 最大塊
   25% 之紙元件，容忍疊紙／折痕）
3. **壓痕定位**（於原始影像偵測：redness>8、CLOSE/OPEN 聚合、分數合併斷段、
   元件紅密度過濾、框外殘紅 >40% 則整紙保守評估）
   ⚠ 偵測必須在白平衡**前**——增益會讓紙面粉霧過門檻、bbox 被撐大
4. **白平衡**（von Kries：紙內非壓痕區中位色 → 235，增益逐張記錄於 CSV）
5. **背景填紙白**（方形裁切超出紙界亦不含背景紙）
6. **旋轉轉正**（minAreaRect 角度，限長寬比 ≥1.15 且 1°–20°）
7. **等比方形裁切**（壓痕長邊佔比 0.834＝論文手動裁切中位數；
   維持原始比例、**無任何拉伸**）
8. **512×512 輸出**＋論文操作點離線預判定（分至 `defect/`｜`good/`）

構圖驗證（vs 論文手動裁切）：長邊 0.865（手動 IQR 0.79–0.89）、短邊 0.719
（手動 0.711）、置中偏移 ≈0、旋轉角四分位 ±1.5°（手動 ±1.7°）。

## 使用流程

```bash
pip install numpy opencv-python pillow   # Python 3.10+
python3 crop_and_pretest_v2.py --src ./photos --out ./review --csv ./預判定報告.csv
```

1. **人工複核** `review/defect|good/` 之檔案歸屬（⚠ **預判定不是真值**，
   判錯的照片直接搬到另一個資料夾）
2. 於[線上工具](../index.html)以「**上傳資料夾**」載入 `review/`
   （路徑含 defect／good 即為真值標註）
3. **參數保持預設**、按「執行」；⚠ **勿按「訓練」**（自動微調會偏離論文操作點）

複核後亦可離線重算指標（與線上工具相互驗證，含 6 種縮放濾鏡穩健性檢查）：

```bash
python3 retest_with_truth.py --review ./review --json ./結果.json --csv ./逐張結果.csv
```

## 注意事項

- 預判定屬**保守設計**：偵測不確定時以整紙評估（偏向判 defect），由人工複核把關。
- 邊界案例（maxBlank 20–24%）對門檻敏感，建議複核時優先放大檢視原圖。
- 本管線於 173 張新批照片（藍紙墊底、冷白平衡）驗證：複核一致性 98.8%
  （FP 2／FN 0，召回率 100%），6 種縮放濾鏡判定完全相同。
