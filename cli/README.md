# CLI 規則式壓痕檢測器 `inspector.py`

本資料夾為論文之**規則式白盒壓痕檢測器**命令列版本，與線上網頁工具（本倉庫 `index.html`）採相同的偵測邏輯。

## 偵測原理（白盒、可解釋）

- 接觸像素 = `(R − (G+B)/2 > redness)` **或** `(亮度 < darkT)`（紅度／暗度雙通道）。
- 48×48 網格 → 內容區（固定 20–80% 邊框）切為**五區**（上／下／左／右／中），計算各區「空白率」。
- 任一區空白率 > 20% 即判 **defect（缺陷）**，否則 **good（良品）**。
- 預設操作點：`redness=18, darkT=130, margin=0.20, frame=0.30, blank=0.20, autoBBox=OFF`。

## 驗證結果（僅以真實影像計）

- 251 張真實壓痕影像：**準確率 98.8%／零漏檢（FN=0）**（瑕疵 223／良品 28）。
- ⚠️ 合成／生成影像不計入準確率。

## 用法

```bash
python3 inspector.py <影像資料夾>              # 批次檢測 + 指標
python3 inspector.py <資料夾> --robust         # 穩健化模式（曝光正規化+限幅去色偏，見下）
python3 inspector.py <資料夾> --deskew         # 旋轉對齊（隨壓痕傾斜校正，見下）
python3 inspector.py <資料夾> --thermal --tim <型號>   # 併 TIM 熱阻/Tcpu 推算
python3 inspector.py <資料夾> --rca --spec     # 根因分析 + SPEC 規格判定
python3 inspector.py --list-tims               # 列出內建 TIM 型錄
```

## `--robust` 穩健化模式

對抗環境光／不同手機色彩／曝光偏移的自適應前處理（兩段式）：

1. **曝光正規化**：以背景亮度均勻增益拉到乾淨基準（`BASE_LUMA≈224`），修正過曝／曝光不足。
2. **限幅去色偏**：偵測紅度時減去「超出乾淨基準（`BASE_RED≈5`）之背景殘留紅度」，單向限幅 `cap=2`。

**對乾淨資料零代價**：robust 與 baseline 於 251 張真實影像之 TP/TN/FP/FN 完全相同（98.8%／零漏檢）。
擾動測試顯示其對「曝光不足／環境偏暗」決定性改善（漏檢歸零），操作邊界約為環境亮度 ≥×0.35。
詳見《擾動穩健性測試報告》。

## `--deskew` 旋轉對齊模式（拍攝傾斜容差）

以接觸塊之最小面積旋轉矩形（OpenCV `minAreaRect`）估壓痕傾斜角，旋轉校正後**仍用固定框**偵測。
實測（251 張真實影像）：**維持零漏檢**，誤檢 3→2、準確率 98.8%→99.2%（修正傾斜良品之誤判）。需安裝 `opencv-python`。

### 框型選擇之實測結論（為何固定框 + 選用旋轉對齊）

| 框型策略 | 準確率 | 瑕疵召回 | 良品 specificity | 判定 |
|---|---|---|---|---|
| 固定 20–80% 正方框（操作點） | 98.8% | 100% | 89.3% | 基準 |
| 自適應伸縮框 `autoBBox` | 88.8% | 100% | **0%** | ❌ 良品全數誤檢 |
| 旋轉對齊 `--deskew` | **99.2%** | 100% | 92.9% | ✅ 採用（選用） |
| 旋轉＋伸縮 | 88.8% | 100% | **0%** | ❌ |

「框隨壓痕外形伸縮（非正方）」會把良品邊緣誤判為缺陷（誤檢暴增）；「置中／平移」會把邊緣缺失移進覆蓋區而新增漏檢。
故 `autoBBox` 維持關閉，操作點採固定框；旋轉對齊為選用之拍攝容差強化。

## 拍攝 SOP（部署建議）

中性白平衡、固定拍攝距離與構圖、避免過暗（≥×0.35）與過曝、關閉手機強化渲染（鮮豔／HDR）。
