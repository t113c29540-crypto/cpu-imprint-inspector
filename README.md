# CPU 壓痕智慧檢測平台（CPU Imprint Inspector）

以富士感壓紙壓痕影像評估 CPU 散熱模組接觸品質的**規則式白盒檢測工具**。
單一 HTML 檔、瀏覽器離線執行、**所有運算與影像皆在本機，不上傳任何資料**。

- **線上使用**：開啟 GitHub Pages 網址即可（亦可下載 `index.html` 離線使用）
- **操作手冊**：[manual.html](./manual.html)

## 方法聲明（與論文一致）
- 判定核心為**規則式白盒分類器**：紅度/亮度雙門檻量化接觸顯色＋上/下/左/右/中五區覆蓋率，任一區空白率 > 20% 判為缺陷；**推論階段不含任何神經網路**。
- 「AI」指以 AI 代理輔助之**門檻校準**（座標下降＋隨機擾動，最小化 1−F1，收斂即停止）；介面之「收斂曲線」為此目標函數之下降過程，**非深度學習訓練損失**。
- **溫度梯度圖／熱流向量為物理模型推估**（T = Tmin + (1−ρ)(Tmax−Tmin)、q = −k∇T 之相對視覺化），**非實測溫度**。
- **TIM 選型推算（v2.2 新增）**：依一階熱阻網路 Rth(TIM) = d/(k·A·C)、Tcpu = Tamb + P·(Rother + Rth) 推算不同 TIM 厚度與導熱係數對應之接觸熱阻與 CPU 溫度；推估 Tcpu 超過警戒值（預設 95 °C）時，自 TIM 型錄（論文附錄四原廠規格）列出滿足客戶規格（預設 Tcpu ≤ 90 °C）之建議。**同為模型推估、非實測**。
- **拍照即時裁切預檢（v2.3／v2.4 新增）**：完整擷取工作流，移植自 `crop_and_pretest_v2.py` 之可行於前端的檢查（白平衡 BNORMAL／壓痕偵測 DETECTION_FAIL／構圖／曝光）：
  - **相機動態預檢（v2.4）**：開啟後鏡頭即時串流，拍照前持續判定品質並疊加置中指引框與偵測框（綠＝壓痕清楚、橙＝待調整）；**異常僅警告、不阻擋**。
  - **拍照／上傳後自動裁切＋首次校準**：自動以壓痕 bbox 推得方形裁切（長邊佔比 0.834）並立即做規則式預判定。
  - **互動裁切編輯器（v2.4）**：可用滑鼠/觸控拖曳移動、拖角縮放裁切框，**反覆「套用裁切並校準」累積結果、可下載滿意的裁切 PNG，直到滿意為止**。
  - 全程本機、影像不外傳；**權威之自動裁切＋預判定仍以 `preprocessing/` 之 Python CLI 為準**。
- 紀錄完整性採 **SHA-256 雜湊鏈（單機 append-only＋輕量 PoW）**：可偵測報告產出後之事後竄改；屬單機機制、非分散式區塊鏈。
- 評估指標：Accuracy / Precision / Recall / F1 / IoU（IoU＝缺陷類 Jaccard＝TP/(TP+FP+FN)，集合層級）。

## 自動前處理管線（v2，2026-06-11）
現場拍攝的「未裁切」原始照片（壓痕非主體、含藍／青色背景紙、白平衡偏冷）不可直接餵給檢測器——構圖與白平衡皆不符演算法前提。[`preprocessing/`](./preprocessing/README.md) 提供與論文 78 張手動裁切構圖對齊之全自動前處理（Python）：

> EXIF 方向校正 → HSV 紙偵測（抗白平衡漂移）→ 壓痕定位（含整紙保守 fallback）→ 紙白 von Kries 白平衡 → 背景填紙白 → **旋轉轉正＋等比方形裁切（壓痕長邊佔比 0.834，無拉伸）** → 512×512 輸出與論文操作點離線預判定

使用流程：前處理 → **人工複核**（預判定不是真值）→ 以「上傳資料夾」載入本工具（參數保持預設、勿按「訓練」）。需求：Python 3.10+、numpy、opencv-python、Pillow。
