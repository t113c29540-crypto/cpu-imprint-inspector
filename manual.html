0. 文件資訊 Document Information

0.1 本手冊用途與讀者 Purpose & Audience

本手冊說明如何操作 CPU 壓痕智慧檢測平台（單一檔案 cpu_imprint_inspector.html）。平台針對 CPU 散熱介面之感壓紙壓痕（Fujifilm Prescale 類）照片，自動判別均勻 (good)／不均勻 (defect)、定位缺失接觸區、產生熱學視覺化與專業檢測報告。

This manual explains how to operate the single-file CPU Imprint Inspector. The platform classifies pressure-sensitive imprint photos of a CPU thermal interface as good/defect, locates missing-contact regions, renders thermal visualizations, and generates a professional report.

讀者對象 Audience本手冊可協助 You will learn to…
品保 / 檢驗人員 QA正確拍攝壓痕、載入影像、判讀 good/defect 與缺失區、產生報告。
製程 / 設備工程師 Process由缺失型態做根因分析 (RCA)、回饋鎖附扭力與安裝對策。
資料 / RD 工程師 Data/RD規劃資料集、調校門檻參數、執行 MCP 自動微調、判讀指標。

1. 系統簡介 System Overview

1.1 這是什麼 What It Is

平台是一個單一 HTML 檔、可離線執行的瀏覽器應用。所有影像分析、指標計算、最佳化與報告產生全部在你的瀏覽器本機完成，不上傳任何資料。介面採護眼深色模式（背景深藍 #1a2b4c、文字淡黃綠 #ccff99），全程繁體中文／English 對照。

核心價值 Core idea壓痕代表 CPU 頂蓋 (IHS) 與散熱器之實際接觸壓力分佈。接觸良好處顯色飽和；空白＝缺失接觸＝熱阻升高＝潛在熱點。系統量測「4 邊框＋中央」共 5 區的空白缺失比例，依 20% 門檻判定。

3. 操作原理 Operating Principles

理解原理能讓你正確設定參數、判讀結果並做根因分析。本章對應平台「1 總覽原理」分頁。

3.1 接觸像素偵測 Contact Pixel Detection

平台把每張影像縮放到 240×240 分析畫布，逐像素判斷是否為「接觸像素」。判據有二，任一成立即視為接觸：

顯色強度 redness = R − (G + B) ÷ 2 ← 感壓紙受壓顯粉紅 亮度 lum = 0.299·R + 0.587·G + 0.114·B 接觸 contact = 1，若 ( redness > 門檻 redness=18 ) 或 ( lum < 暗度門檻 darkT=130 ) = 0，否則
為何用兩個判據粉紅顯色（redness）抓住正常受壓區；暗度（darkness）補抓深色／陰影接觸，降低漏判。門檻 redness 可在各分頁的「③ 參數與目標」調整。

3.2 接觸密度網格 Contact Density Grid

畫布被切成 48×48 的網格。每個網格的接觸密度 = 該格內接觸像素數 ÷ 像素總數（值介於 0~1）。此密度圖即後續區域量測、缺失定位與溫度場的基礎。

密度高＝接觸良好 密度低＝接觸缺失（熱點候選）

3.3 五區量測法 The Five-Region Method

平台在壓痕內容區（由 margin 邊距界定，或開啟 autoBBox 自動偵測範圍）切出 5 個量測區，邊框寬度由 frame 比例決定：

區域 Region位置常見缺失成因（速覽）
top 上框內容區上緣邊框帶安裝傾斜、單邊壓力不足、立柱高度不一、IHS 翹曲。↗ §9
bottom 下框內容區下緣邊框帶
left 左框內容區左緣邊框帶
right 右框內容區右緣邊框帶
center 中央四邊框內的中央方塊TIM 量不足、底面中央凹陷、泵出效應。

每區計算覆蓋率 Coverage（該區平均接觸密度）與空白率 Blank：

Coverage(r) = 區域 r 的平均接觸密度 Blank(r) = 1 − Coverage(r)

3.4 判定規則 Decision Rule（20% 門檻）

maxBlank = max[ Blank(top), Blank(bottom), Blank(left), Blank(right), Blank(center) ] 判定 Rule： 若 maxBlank > 空白門檻 blank(=20%) ⇒ defect 不均勻 否則 ⇒ good 均勻

意即：只要任何一區的空白缺失超過 20%，整張即判為 defect。這是嚴格的「最差區」邏輯，符合品保「任何接觸不良都是風險」的精神。門檻 blank 預設 0.20，可在「③ 參數與目標」調整。

先天限制 Inherent limit純「裸紙、完全無接觸」因感壓紙暖色與極淡接觸在色彩上重疊，無法穩定自動區分。因此 redness 預設 18 是用來抓「接觸弱或缺失」的可重現門檻，而非區分裸紙。

3.5 評估指標 Evaluation Metrics

以缺陷 defect 為正類 (positive)。TP 真陽／TN 真陰／FP 偽陽／FN 偽陰。

混淆矩陣 Confusion Matrix 實際 good 實際 defect 預測 good TN FN 預測 defect FP TP
準確率 Accuracy = (TP + TN) ÷ (TP + TN + FP + FN) 目標 > 95% 精確率 Precision = TP ÷ (TP + FP) 目標 > 95% 召回率 Recall = TP ÷ (TP + FN) 目標 > 97% F1 分數 F1-Score = 2 × (Precision × Recall) ÷ (Precision + Recall) 目標 > 96% 交並比 IoU = TP ÷ (TP + FP + FN) 目標 > 75%
召回率最重要目標將 Recall 設到 97%（高於其他指標）：在品保情境，漏掉一個缺陷 (FN) 比誤殺一個良品 (FP) 代價更高，故優先壓低 FN。判讀指南見 ↗ §7。

3.6 熱學視覺化 Thermal Model（示意 illustrative）

將接觸密度反推為示意溫度場，缺失區＝高溫熱點。

溫度場 T(x,y) = Tmin + ( 1 − 接觸密度 ρ(x,y) ) × ( Tmax − Tmin ) 單位 °C （量化到整數度，每 1°C 一色，附色階尺） 傅立葉熱傳導 Fourier： 熱流密度向量 q = − k × ∇T 單位 W/m² 其中 ∇T = ( ∂T/∂x , ∂T/∂y ) ， k = 等效熱傳導係數

預設 Tmin=40°C、Tmax=95°C、k=5。在報告中以彩色梯度圖呈現溫度場，並疊加箭頭表示熱流方向與大小。↗ §8.3

符號相容性平台已全面移除 LaTeX，所有數學以 × ÷ ∩ ∪ ≥ ≤ ° Σ √ ∇ 等標準 Unicode 呈現，確保任何瀏覽器／PDF 皆不亂碼。

4. 拍攝與資料準備 Capture & Data Preparation

本章對應「2 資料集」分頁。良好的輸入是準確判定的前提。

4.1 拍攝七要點 Capture Guidelines

  1. 拍攝模式選「正方形」 Square mode相機比例設為 1:1，避免長寬變形影響後續區域量測。
  2. 壓痕轉正、置中 Rotate & center壓痕角度轉正、對齊畫面正中央。
  3. 四邊預留 20% 留白 20% margin壓痕四邊與畫面邊界各預留 20% 留白（對應 margin=0.20）。
  4. 原始檔 PNG 匯出 Export PNG以原始 PNG 匯出，保留真實資訊（不壓縮失真、不轉檔）。
  5. 放入 Input、不自動裁剪、用原圖 No auto-crop照片放入指定資料夾；分析一律使用原圖。
  6. 缺陷判準 Defect4 邊框與中央，空白缺失 > 20% 接觸面積 ⇒ defect。
  7. 良品判準 Good4 邊框與中央，空白缺失 ≤ 20% 接觸面積 ⇒ good。
穩定拍攝小技巧固定光源與距離、避免反光與陰影、感壓紙平貼無皺褶、同一批次用相同相機設定 — 可大幅提升判定一致性。

4.2 良品 vs 缺陷 定義 Good vs Defect

均勻壓痕 GOOD

4 邊框與中央區域，空白缺失 ≤ 20%。顯色均勻飽和、無明顯缺口。

不均勻壓痕 DEFECT

4 邊框與中央區域，空白缺失 > 20%。可見角落／單邊／中央缺口（接觸不良＝熱點）。

5.2 校準集操作 Training（七步驟）

  1. ① 批次載入影像 Batch load 三種來源任選其一：
    • 📁 選擇資料夾：一次載入整個資料夾，依資料夾名自動標註 good/defect（建議）。
    • 🖼️ 選擇多個檔案：手動多選影像檔。
    • ✨ 載入示範樣本：12 張合成樣本，供快速試用。
    載入後會顯示張數與已標註數（good X / defect Y）。僅接受影像檔（png/jpg/webp/bmp）。
  2. ② 硬體與模型 Hardware & Model 檢視自動偵測的硬體資訊與推薦等級。設定：
    • 部署方式：「本機自動下載安裝」或「雲端 API Key」（選雲端時填入 sk-...）。
    • 辨識模型：由前 5 名下拉選擇（依硬體排序）。
    • 生成增強模型：由前 5 名下拉選擇（資料增強用）。
    可展開「前 5 名模型清單」查看規格與來源。↗ §附錄 C
  3. ③ 參數與目標 Parameters 設定 4 個核心參數：Redness T、Margin、Frame、Blank，可勾選「自動偵測壓痕範圍 autoBBox」。目標值固定顯示於下方。詳見 ↗ §6 參數調校。
  4. ④ 執行 MCP 自動微調 Run 設定最大迭代 Max iter（預設 300，上限 1000），按 ▶ 開始 校準 Cal。
    • 平台以「座標下降 + 隨機擾動」搜尋最佳門檻，損失 = 1 − F1，逐步接受更優參數。
    • 收斂即停止 (early stopping)：5 項指標全部超過目標即停止。
    • 進度條與狀態列即時顯示 iter k/N、F1、Acc。
  5. ⑤ 判讀 收斂曲線（1−F1） 橙線＝損失 Loss (1−F1) 應下降；綠線＝準確率 Accuracy 應上升。橫軸為迭代次數。
  6. ⑥ 判讀指標 Metrics 5 張 KPI 卡片顯示各指標數值與是否達標（✓/✗），下方列出混淆矩陣 TP/TN/FP/FN 與已標註樣本數。判讀見 ↗ §7。
  7. ⑦ 結果縮圖 & 儲存模型 檢視結果縮圖（每張標 good/defect 與 maxBlank，紅框＝缺失區）。確認無誤後按 💾 儲存模型，下載 cpu_cal_yymmddhhmm.pth。此檔名會自動填入「驗證集」分頁的依據模型欄位。
未標註時 MCP 無法真正最佳化若已標註樣本 少於 2 張，平台無法計算真實指標，僅顯示「示範收斂曲線」並提示 Unlabeled: demo curve。要真正微調，請確保載入的影像帶有 good/defect 真值。

5.3 驗證集操作 Validation

切到 4 驗證集。介面與步驟同 §5.2，差異如下：

  • 分頁頂端顯示「依校準模型 cpu_cal_…….pth 進行驗證」（由 §5.2 儲存時自動帶入）。
  • 載入驗證子集影像（val_good / val_defect）。
  • 執行後若達標，儲存模型輸出 cpu_val_yymmddhhmm.pth，並自動帶入「測試集」分頁。
目的驗證集用於確認在校準階段調出的參數，在未直接參與調校的資料上仍維持指標，避免過度貼合 (overfitting)。

5.4 測試集操作 Test

切到 5 測試集。介面與步驟同 §5.2，差異如下：

  • 分頁頂端顯示「依驗證模型 cpu_val_…….pth 進行測試」。
  • 載入測試子集影像（test_good / test_defect）。
  • 測試集應為最終、未曾用於調校的資料，其指標最能代表真實效能。
  • 儲存模型輸出 cpu_test_yymmddhhmm.pth。
真值如何提供（good／defect 怎麼來） 本工具的「真值標籤」取自檔案路徑字串（資料夾名＋檔名）：路徑含 defect／bad／ng 判為缺陷、含 good／ok 判為良品。建議用「選資料夾」方式載入、並將影像分放 test_good／test_defect 子資料夾——資料夾名即會被當路徑一部分提供真值。 若改用「逐檔選取」，則只看檔名：檔名須自帶 good／defect 關鍵字（如 test_good_001.png）才會被標註。 若資料夾名與檔名都不含關鍵字（如 IMG_7958.png），該影像視為「未標註」，此時仍可得到 good／defect 判定，但無法計算準確率、召回率與混淆矩陣（無真值可比對）；MCP 門檻校準亦需有真值方能進行。

5.5 一鍵載入資料集（含真值） One-Click Dataset Load

若已整理好 cpu_imprints_dataset/ 並以本機伺服器開啟（↗ §2.2 方式 B），可在 2 資料集 分頁一次完成全流程：

  1. 載入按 📂 載入 cpu_imprints_dataset/（含真值）。平台讀取 filename_mapping.csv，依路徑解析 split 與 good/defect 真值，逐張分析。
  2. 檢視即時指標載入完成後直接顯示三子集分佈與對真值的 5 項指標（含 TP/TN/FP/FN）。
  3. 產生報告按 📑 產生含真值指標報告，自動跳到「報告」分頁並建立完整報告。
  4. 匯出 CSV按 ⬇ 匯出結果 CSV，輸出 cpu_dataset_yymmddhhmm.csv（含 truth / prediction / correct / 各區覆蓋率）。
自我一致性警告若 CSV 的標註來自模型自動預標，指標會趨近 100%（自我一致性），非真實泛化效能。人工校正後再以「📁 選擇資料夾」重載，數字才有效。
file:// 無法使用本功能直接雙擊開檔時此按鈕會報「找不到 CSV／畫布污染」。請改用本機伺服器，或改用各分頁「📁 選擇資料夾」（不受限制）。

7.1 KPI 卡片怎麼看 Reading KPI Cards

7.2 混淆矩陣與品保意義 Confusion Matrix in QC Terms

符號意義品保情境
TP 真陽實際 defect，預測 defect✅ 正確攔截缺陷
TN 真陰實際 good，預測 good✅ 正確放行良品
FP 偽陽實際 good，預測 defect⚠️ 誤殺良品（過殺，影響良率）
FN 偽陰實際 defect，預測 good🔴 漏判缺陷（最危險，流出風險）

7.3 指標—行動對照 Metric → Action

指標偏低代表下一步
召回率 Recall 低FN 多，漏判缺陷最優先處理：調高敏感度（§6.4），複檢被判 good 的樣本。
精確率 Precision 低FP 多，過殺良品放寬門檻，檢視被判 defect 的良品是否拍攝/標註問題。
F1 低Precision 與 Recall 失衡讓 MCP 重新搜尋；或檢查資料平衡。
IoU 低預測與真實重疊不足多半伴隨整體判別差，回頭檢查標註與門檻。
Accuracy 高但 Recall 低類別不平衡假象勿只看 Accuracy；以 Recall / F1 為準。

7.4 何時指標顯示 N/A？ When Metrics Are N/A

  • 影像無 good/defect 真值（檔名/資料夾未含關鍵字）⇒ 5 項指標無法計算，報告改顯示「缺陷率／良品率」。
  • 已標註樣本少於 2 張⇒ 無法 MCP 最佳化，僅顯示示範收斂曲線。
趨近 100% 要警覺指標異常完美（接近 100%）通常代表標註是模型自動預標（自我一致性），而非真實效能。務必人工校正後重載。↗ §10

8b. TIM 選型推算 TIM Thermal Estimation

本章對應 7 TIM選型 分頁。依不同 TIM 之厚度 d 與導熱係數 k 推算接觸熱阻 Rth 與 CPU 溫度 Tcpu；推估超過警戒值時自動列出滿足客戶規格之 TIM 建議。全部為一階模型推估，非實測。

8b.1 模型 Model（論文式 3.5、3.6）

Rth(TIM) = d ÷ (k × A × C)　　　…(3.5 之 TIM 項，Aeff = A × C)
Tcpu = Tamb + P × (Rother + Rth(TIM))　…(3.6)
  • d＝TIM 厚度/接合線厚度 BLT（mm）、k＝導熱係數（W/m·K）。
  • C＝接觸覆蓋率（1−整體空白率，0–1）：可手動設定情境，或一鍵帶入已載入影像之最差／最佳樣本。
  • A＝名義接觸面積（mm²）、P＝CPU 功耗（W）、Tamb＝環境溫度（°C）、Rother＝TIM 以外之總熱阻（°C/W，含晶粒內部與散熱器側）。
預設值出處P＝40 W 與 Rother＝0.80 °C/W 依論文 §4.10 之 PTL 40 W 級實測平台推估；A＝200 mm² 取論文表 2.1 晶粒面積量級；警戒 95 °C＝熱場視覺化上限，客戶規格 Tcpu ≤ 90 °C。皆可於「7.1 推算參數」調整。

8b.2 操作 Operation

  1. 於 7.1 選擇現行 TIM（內建型錄含 T-Global／Honeywell／Thermal Grizzly／Arctic 等，數值取自論文附錄四原廠規格；亦可自訂 k、d）。
  2. 於 7.2 設定覆蓋率 C：拖曳滑桿做情境分析（例：50%＝論文表 4.4 重度缺陷），或按 ⤓ 取載入影像之最差樣本 帶入實際量測值。
  3. 7.3 即時顯示 Rth(TIM)、Rth(total) 與 Tcpu，並標示判定：✓ 達標（≤ 客戶規格）／△ 介於規格與警戒間／⚠ 超過警戒。
  4. Tcpu 超過警戒值時，7.4 自動以同條件重算全型錄並依溫度升冪排序，✓＝滿足 Tcpu ≤ 客戶規格之建議選項。
判讀注意C＜20% 屬一階模型外插範圍，絕對溫度僅供相對比較；此情形應優先依 RCA（§9）改善接觸根因（鎖附扭力、安裝、TIM 量），再以高 k、低 BLT 之 TIM 補強。液態金屬導電且腐蝕鋁，須評估風險。實際選型請以最新原廠規格書與實測驗證為準。

10. 疑難排解 Troubleshooting

「找不到 filename_mapping.csv／無法讀取像素」

原因：以 file:// 雙擊開檔，瀏覽器禁止讀取本機檔與已污染畫布。
解法：① 改用本機伺服器（§2.2 方式 B）；或 ② 改用各分頁「📁 選擇資料夾」載入（不受限制，仍依資料夾名取得真值）。

指標趨近 100%，好得不真實

原因：標註來自模型自動預標，指標只反映自我一致性。
解法：人工把錯標檔案移到正確的 good/defect 資料夾，再用「📁 選擇資料夾」重新載入。↗ §4.6、§7.4

指標顯示 N/A 或只看到缺陷率

原因：影像沒有 good/defect 真值（檔名/資料夾缺關鍵字）。
解法：依 §4.3 命名規範整理檔名與資料夾，含 good/defect（或 ok/ng/bad）字樣。

按「開始」後只顯示「示範收斂曲線」

原因：已標註樣本少於 2 張，無法計算真實指標。
解法：載入帶真值的影像（至少 2 張、建議 good/defect 皆有）。

未完全達標（已達迭代上限）

原因：在搜尋上限內 5 項指標未全數達標。平台會保留最佳 F1 的參數。
解法：提高 Max iter（上限 1000）、改善拍攝一致性、校正標註、增加樣本，或依 §6.4 手動微調。

匯出的 PDF 變成白底／缺少色塊

原因：列印對話框未啟用背景圖形。
解法：在列印設定中勾選「背景圖形 / Background graphics」，即可保留深色主題與彩色圖。

裸紙（完全無接觸）判定不穩

原因：感壓紙暖色與極淡接觸在色彩上重疊，無法穩定自動區分純裸紙。
解法：這是方法先天限制。請確保拍攝的是有受壓壓痕的感壓紙；裸紙不應納入待測。↗ §3.4

環境晶片顯示等級 LOW

原因：偵測到核心數/記憶體較低或無 WebGPU。
影響：不影響本平台分析（皆為輕量運算）；僅影響「推薦模型排序」。可正常使用所有功能。

11. 常見問題 FAQ

Q1. 我的影像會被上傳嗎？

不會。所有處理皆在你的瀏覽器本機完成，不上傳任何資料。

Q2. 下載的 .pth 是真的 PyTorch 權重嗎？

不是 PyTorch 張量權重。它是瀏覽器校準的門檻型參數（JSON 內容，含已學參數、硬體、目標與指標），以 .pth 命名以符合工作流規格。↗ §1.4

Q3. 需要網路或安裝任何東西嗎？

基本操作不需要，可完全離線。只有「一鍵載入資料集」需要一個本機伺服器來讀取檔案。

Q4. 為何召回率目標 (97%) 比其他指標高？

品保上「漏判缺陷 (FN)」風險最高，故優先壓低 FN、拉高 Recall。↗ §3.5、§7

Q5. 一定要分校準/驗證/測試三步嗎？

若只是快速判定一批照片，可只在單一分頁載入並執行，再產生報告。但要評估泛化效能，建議完整走三階段並以測試集為準。

Q6. 可以一次處理幾張？

沒有硬性上限；結果縮圖最多顯示 60 張、報告三聯圖會渲染全部影像。張數越多渲染越久，平台會分批讓出畫面避免卡頓。

Q7. 平台支援哪些影像格式？

PNG / JPG / JPEG / WEBP / BMP。拍攝建議用原始 PNG 以保留資訊。

12. 安全、隱私與合規 Security, Privacy & Compliance

  • 本機運算 Local-only：影像與結果不離開瀏覽器，杜絕外洩。
  • 完整性 Integrity：雜湊鏈防竄改，任一筆遭改即驗證失敗。
  • 機密性 Confidentiality：AES-GCM 256 位元加密（PBKDF2 金鑰派生）保護匯出摘要。
  • 稽核 Audit：所有操作寫入稽核軌跡 audit log，可於報告展開檢視。
  • 輸入驗證 Input validation：僅接受影像檔，防止惡意檔注入。
