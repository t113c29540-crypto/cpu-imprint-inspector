一、系統總覽與原理 System Overview & Principle

本平台針對 CPU 散熱介面感壓紙壓痕（Fujifilm Prescale 類）影像，進行均勻 (good)／不均勻 (defect) 之自動判別、缺失定位、熱學視覺化與報告產生。整體採護眼深色模式（背景深藍 #1a2b4c、文字淡黃綠 #ccff99），全程繁中／English 對照。

This platform classifies pressure-sensitive imprint photos of a CPU thermal interface into good (uniform) / defect (non-uniform), locates missing-contact regions, renders thermal visualizations, and generates a professional report.

判定原理 Decision principle：壓痕代表 CPU 頂蓋 (IHS) 與散熱器之實際接觸壓力分佈。接觸良好處顯色飽和；空白＝缺失接觸＝熱阻升高＝潛在熱點。本系統量測「4 個邊框＋中央」共 5 區的空白缺失比例，依 20% 門檻判定。
工具資源 Resources： 📖 操作手冊 manual.html　· 🐍 自動前處理 crop_and_pretest_v2.py　· 🐍 複核後離線重測 retest_with_truth.py
現場拍攝之「未裁切」原始照片（壓痕非主體、含藍／青背景紙、白平衡偏冷）請先以 自動前處理腳本裁切對齊論文構圖——EXIF 校正、紙偵測、紙白白平衡、旋轉轉正＋等比方形（無拉伸）、512×512 輸出與離線預判定 （需 Python 3.10+／numpy／opencv-python／Pillow）。 人工複核 defect／good 分夾（預判定不是真值）後，再以「上傳資料夾」載入本工具： 參數保持預設按「執行」，勿按「校準」；複核後亦可用離線重測腳本重算指標（含 6 種縮放濾鏡穩健性驗證）與本工具相互對照。

1.2 評估指標公式 Evaluation Metrics（標準文字符號，非 LaTeX）

定義：缺陷 defect 為正類 (positive)。TP 真陽／TN 真陰／FP 偽陽／FN 偽陰。N = 總樣本數。

混淆矩陣 Confusion Matrix 實際 good 實際 defect 預測 good TN FN 預測 defect FP TP
總體準確率 Accuracy = (TP + TN) ÷ (TP + TN + FP + FN) 目標 Target > 95% 精確率 Precision = TP ÷ (TP + FP) 目標 Target > 95% 召回率 Recall = TP ÷ (TP + FN) 目標 Target > 97% F1 分數 F1-Score = 2 × (Precision × Recall) ÷ (Precision + Recall) 目標 > 96% 交並比 IoU（缺陷類 Jaccard）= TP ÷ (TP + FP + FN) (= 預測 ∩ 真實 ÷ 預測 ∪ 真實) 目標 > 75%
區域空白缺失率 Region blank ratio Blank(r) = 1 − Coverage(r) = 1 − (接觸像素 ÷ 區域總像素) 判定 Rule：若 任一區 max[ Blank(top,bottom,left,right,center) ] > 20% ⇒ defect，否則 good
熱學視覺化 Thermal model（示意 illustrative） 溫度場 T(x,y) = Tmin + (1 − 接觸密度 ρ(x,y)) × (Tmax − Tmin) 單位 °C 傅立葉熱傳導 Fourier: 熱流密度向量 q = −k × ∇T 單位 W/m² 其中 ∇T = ( ∂T/∂x , ∂T/∂y )， k = 等效熱傳導係數 thermal conductivity

1.3 技術誠實聲明 Honesty Note

  • 真實運算 Real：影像接觸像素偵測、5 區覆蓋率、混淆矩陣與 5 項指標、門檻參數最佳化搜尋（MCP）、溫度梯度圖（模型推估）、熱流向量、ControlNet 風格缺失標註，皆於瀏覽器實際計算。
  • 流程展示 Demonstrated：深度 CNN/U-Net 之 GPU 訓練與二進位 .pth 權重無法於瀏覽器產生；本平台以「門檻型形態學分類器」實作真實學習，並依規格命名下載模型檔（內含已學參數＋中繼資料）。深度學習工作流以動畫與說明完整呈現。
  • 本機隱私 Privacy：所有影像僅在你的瀏覽器內處理，不上傳。All processing is local; no upload.

2.2 良品／缺陷 定義 Good vs Defect

均勻壓痕 GOOD

4 個邊框與中央區域，空白缺失 ≤ 20% 接觸面積。顯色均勻飽和、無明顯缺口。

不均勻壓痕 DEFECT

4 個邊框與中央區域，空白缺失 > 20% 接觸面積。可見角落／單邊／中央缺口（接觸不良＝熱點）。

兩張示意圖由程式即時繪製並標出 5 個量測區（top／bottom／left／right／center）。紅框=缺失區 missing region。
判定門檻 redness=18：偵測「接觸弱或缺失」（任一區空白>20%⇒defect）。註：純「裸紙、完全無接觸」因感壓紙暖色與極淡接觸在色彩上重疊，無法穩定自動區分，故採此可重現門檻。可於各分頁微調。

七、TIM 選型推算 TIM Thermal Estimation模型推估・非實測 Model estimate, not measured

依一階熱阻網路推算不同 TIM 厚度 d 與導熱係數 k 對應之接觸熱阻與 CPU 溫度（論文式 3.5、3.6）： Rth(TIM) = d ÷ (k × A × C)、Tcpu = Tamb + P × (Rother + Rth(TIM))， 其中 C＝接觸覆蓋率（1−整體空白率，可由壓痕影像量測帶入）、Aeff＝A×C。 推估 Tcpu 超過警戒值時，自 TIM 型錄（論文附錄四 表附四-1／附四-3 原廠規格）重算各候選，凡滿足客戶規格者依溫度升冪列為建議。
